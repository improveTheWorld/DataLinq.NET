# AI Context Management - Setup Instructions

This folder contains everything needed to set up the AI context management system in a new repository.

## Files Included

| File | Purpose |
|------|---------|
| `context-management.md` | Rules file (persistent AI guidance) |
| `scan.md` | `/scan` workflow - Full context generation |
| `refresh.md` | `/refresh` workflow - Read context for current task |
| `update.md` | `/update` workflow - Add new info locally |

---

## Installation Instructions

**When a user asks you to set up AI context management in a repository:**

### Step 1: Create the folder structure

```
[repo-root]/
└── .agent/
    ├── workflows/
    └── rules/
```

### Step 2: Copy workflow files

Copy these files to `.agent/workflows/`:
- `scan.md`
- `refresh.md`
- `update.md`

### Step 3: Copy rules file

Copy `context-management.md` to `.agent/rules/`

### Step 4: Run initial scan

Execute `/scan` to generate the initial context index for the repository.

---

## Post-Installation Structure

After setup, the repository will have:

```
.agent/
├── index.md                    # Generated by /scan
├── context/                    # Generated by /scan
│   ├── architecture.md
│   ├── source.md
│   ├── docs.md
│   ├── tests.md
│   └── packaging.md
├── workflows/
│   ├── scan.md
│   ├── refresh.md
│   └── update.md
└── rules/
    └── context-management.md
```

---

## Usage After Setup

| Command | When to Use |
|---------|-------------|
| `/refresh` | Start of conversation, or to reload context for current task |
| `/update` | After learning new information that should be persisted |
| `/scan` | To regenerate all context files from scratch |

The rules file ensures the AI automatically:
- Runs `/refresh` at conversation start
- Runs `/update` silently when corrected about something
- Checks indexed context before exploring the codebase
