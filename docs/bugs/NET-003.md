## Description
Originally reported: when `ObjectMaterializer.Create<T>()` is called with a type that has no parameterless constructor, the error message does not include helpful context.

## Resolution
**Not a bug.** `CreateViaPrimaryConstructorWithSchema<T>()` correctly handles types without parameterless constructors by mapping schema names to constructor parameters (case-insensitive). The test was written assuming the materializer couldn't handle this case, but it can.

Example: `NoDefaultConstructor(string name)` + schema `["Name"]` â†’ parameter `name` matched â†’ object created successfully.
