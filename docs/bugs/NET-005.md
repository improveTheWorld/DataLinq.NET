## Description
The schema dictionary uses `StringComparer.OrdinalIgnoreCase`, meaning columns `Name`, `name`, and `NAME` all map to the same key. This prevents mapping to models with case-variant properties (e.g., `Name`, `name`, `NAME` as separate properties).

## Expected vs Actual
- **Expected:** `Name="Title Case"`, `name="lower case"`, `NAME="UPPER CASE"`
- **Actual:** All three map to the same key; last value wins (`Name="UPPER CASE"`)

## Notes
This is by design (case-insensitive matching is convenient for CSV). Consider adding an option for case-sensitive mode.
